LOAD A MODULE AT BOOT
________________________________________________________________________________

NOTE: Enable the udevd service if you opt to load a module at boot as the
      daemon is needed to setup whatever drivers you load.

      SEE: @/init/busybox


Using /etc/inittab (busybox-init)
________________________________________________________________________________

A module can be loaded at boot by adding a single line to your /etc/inittab.

+------------------------------------------------------------------------------+
|                                                                              |
|   ::once:/bin/modprobe <module name>                                         |
|                                                                              |
+------------------------------------------------------------------------------+

Using /etc/rc.d (all inits)
________________________________________________________________________________

A module can be loaded at boot by adding a boot hook to your /etc/rc.d folder.
The file must end with `*.boot`. For example you might add the following in
/etc/rc.d/load-modules.boot

+------------------------------------------------------------------------------+
|                                                                              |
|   /bin/modprobe <module name>                                                |
|                                                                              |
+------------------------------------------------------------------------------+
